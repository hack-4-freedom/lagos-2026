# Bitscy

**Hack4Freedom Lagos 2026**

---

## Overview

Bitscy is a Bitcoin-native marketplace for African women artisans, makers whose handmade goods were locked out of global commerce. Sellers list their craft, buyers pay with Lightning from anywhere on earth, and sellers withdraw to their local bank accounts.

The product is a mobile-first PWA, installable to any Android or iOS home screen, no app stores. Identity is a Nostr keypair generated and encrypted client-side, so the server never sees a password or plaintext key. Products and orders are mirrored to public Nostr relays as NIP-15 events, so the catalog lives on even if Bitscy disappears.

---

## Problem

Talented African women artisans are structurally excluded from global commerce:

- **Etsy** doesn't cleanly support Nigerian sellers.
- **PayPal** blocks Nigerian accounts.
- **Stripe** isn't available in Nigeria.
- International buyers want to buy their handmade work directly but lack a frictionless way to pay them.
- Identity systems that require KYC, residential addresses, or established credit shut out artisans who don't have those things.

The result: a Nigerian artisan who could sell to a buyer in Toronto can't, because no rail connects them.

Bitcoin Lightning is the only payment system that genuinely doesn't care about nationality, banking jurisdiction, or platform policies. Bitscy turns that property into a usable marketplace.

---

## Solution

Three core pieces:

**1. Lightning-native payments.** Buyers anywhere pay a BOLT-11 invoice generated by a platform-managed Breez wallet. Sellers see balance updates instantly. No bank account required to receive payment.

**2. Nigerian bank withdrawals via Bitnob.** Sellers withdraw sats to their Nigerian bank in NGN within minutes. v1 runs on Bitnob's sandbox (real API, real Lightning routing, no real NGN moves) with an honest production path post-KYB.

**3. Nostr identity, zero-knowledge auth.** Sellers create an account with just a shop name and password. No email, no phone, no KYC. The Nostr keypair is generated in the browser, the 12-word recovery mnemonic is encrypted client-side with PBKDF2 + AES-GCM, and only the encrypted blob ships to the server. Products (NIP-15 kind 30018) and orders (kind 30019) are signed and published to public Nostr relays.

## Technology Stack

**Frontend**

- Next.js 16 (App Router) + React 19
- TypeScript (strict)
- Tailwind CSS 4
- Zustand for client-side state
- Serwist for the PWA service worker
- qrcode.react for Lightning invoice QR

**Backend**

- Next.js API Routes
- Prisma 6 + PostgreSQL (Supabase)
- Cloudinary for image uploads (signed direct upload, files never touch our server)

**Bitcoin / Lightning**

- Breez SDK Liquid (platform wallet, signet for dev / mainnet for demo)
- Bitnob API for NGN off-ramp (sandbox in v1)
- CoinGecko for live BTC/NGN exchange rate

**Nostr (standard NIPs)**

- nostr-tools
- NIP-01: basic protocol, profile (kind 0)
- NIP-04: encrypted shipping addresses (buyer → seller)
- NIP-06: BIP39 mnemonic → Nostr secret key
- NIP-15: marketplace events (kind 30018 products, kind 30019 orders)
- NIP-98: HTTP auth via kind 27235 challenge signing

**Hosting**

- Vercel (frontend + API routes)
- Supabase (PostgreSQL)

## Status

Working end-to-end on the Bitscy live site:

- Zero-knowledge signup / signin / session for both buyers and sellers (no password leaves the client)
- Seller profile editing with Cloudinary avatar upload
- Public marketplace browse grid with category filter
- Product CRUD: create with multi-image upload, list, edit, unlist / restore, public detail page
- Lightning checkout flow: order creation → BOLT-11 QR → 15-minute countdown → automatic polling → settlement → success screen
- Backend Lightning settlement via Breez SDK Liquid
- User-visible order IDs in `BTS-XXXX-XXXX` Crockford Base32 format (no I/L/O/U), readable, copyable, easy to repeat over the phone
- Per-seller ledger with reconciliation endpoint
- Seller dashboard with real balance, total sales, total earned, recent orders (all from ledger + orders APIs)
- Seller order management: incoming orders list with status filter, order detail with timeline, inline mark-as-shipped action wired to `PATCH /api/orders/[id]/ship`
- Client-side NIP-04 decryption path for shipping addresses on the seller order detail page (renders structured address once buyer-side shipping form lands)
- Buyer order history: list page with status pills + detail page with timeline, seller summary, and reference copy
- Bank account management: list saved Nigerian bank accounts, add new account inline during withdraw, masked display (last 4 digits)
- Seller withdraw flow: real balance, bank picker, NGN → sats conversion, submit to `/api/payout`, success view with the Bitnob payout ID and ETA
- Withdraw history page: paginated payouts with real status pills, masked bank label, copyable reference
- Bitnob sandbox integration for NGN payouts (frontend + backend wired)
